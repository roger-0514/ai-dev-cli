# Node CLI 7天学习清单（AI全栈转型）

## 目标

7 天内做出一个可展示的 `ai-dev-cli` 雏形，并具备基础工程化能力。

## Day 1：搭骨架（可运行）

- 目标：CLI 能启动并识别子命令。
- 任务：
  - 初始化 TypeScript 项目（建议 `pnpm`）。
  - 设计命令：`init` / `ask` / `summarize`。
  - 跑通 `--help`、`--version`。
- 验收：
  - `ai-dev-cli --help` 正常显示。
  - 子命令有占位输出。

## Day 2：参数与配置（可用）

- 目标：支持参数优先级和配置文件。
- 任务：
  - 支持 `--model`、`--temperature`、`--json`。
  - 加载 `~/.ai-dev-cli/config.json`。
  - 规则：CLI 参数 > 环境变量 > 配置文件 > 默认值。
- 验收：
  - 同一参数从不同来源输入，结果符合优先级。

## Day 3：日志与错误处理（可靠）

- 目标：把“能跑”升级为“可维护”。
- 任务：
  - 日志分级：info / warn / error / debug。
  - 统一错误码（如 `E_CONFIG`、`E_NETWORK`、`E_TIMEOUT`）。
  - 进程退出码规范（成功 0，失败非 0）。
- 验收：
  - 故意制造错误时，提示清晰且退出码正确。

## Day 4：Mock AI 能力（功能闭环）

- 目标：不接真实模型，先跑通业务链路。
- 任务：
  - `ask`：输入问题返回 mock 回答。
  - `summarize`：读取文本后返回摘要（mock）。
  - 支持 `--json` 输出结构化结果。
- 验收：  
  - 两个命令都可用，且文本/JSON 两种格式可切换。

## Day 5：测试（质量基础）

- 目标：给核心行为加自动化保障。
- 任务：
  - 使用 vitest 测试框架
  - 测试参数解析。
  - 测试配置优先级。
  - 测试错误场景（缺参数、配置非法）。
- 验收：
  - 核心测试通过，覆盖关键路径（不必追求全覆盖）。

## Day 6：接真实 LLM（最小接入）

- 目标：从 mock 升级到真实调用。
- 任务：
  - 封装 provider 层（先支持一个模型）。
  - 加入超时、重试（1-2 次）和错误分类。
  - 输出 token/耗时统计（先粗粒度）。
- 验收：
  - `ask` 可拿到真实回复。
  - 异常时能重试并给友好错误提示。

## Day 7：打包与作品化（可展示）

- 目标：形成可演示项目。
- 任务：
  - 补 README（安装、命令、示例、常见错误）。
  - 录 2-3 分钟演示（终端操作）。
  - 写一页复盘：做了什么、踩了什么坑、下一步是什么。
- 验收：
  - 新环境按 README 能跑通。
  - 有可分享的 demo 与项目说明。

## 每天固定流程（建议）

- 20 分钟：看概念（只看当天用得到的）。
- 90 分钟：编码。
- 20 分钟：测试 + 修复。
- 15 分钟：写当日复盘（学到什么 / 问题 / 明天计划）。

## 常见坑提醒

- 一上来做“大而全”架构。
- 过早引入太多库，导致复杂度失控。
- 只看教程不产出代码。
- 忽略错误处理和退出码。

## 7天结束后的能力标准

- 能独立完成一个工程化 CLI（不是脚本拼凑）。
- 具备最小测试和错误治理意识。
- 能把 mock 能力替换成真实模型调用。
